# Security Policy

## Supported Versions

Only the latest pre-release/release version of Verbatim is actively supported.

| Version | Supported           |
| ------- | ------------------- |
| < 1.0   | Latest version only |

## Reporting a Vulnerability

If you discover a potential security vulnerability in Verbatim, please report it immediately:

1. **Private Vulnerability Reporting**: If possible, use GitHub's native Private Vulnerability Reporting feature by visiting the repository's "Security" tab and choosing "Report a vulnerability".
1. **Email Fallback**: If Private Vulnerability Reporting is unavailable, please email Karl Johnson at `karl.johnson@example.com` with the details. Do not open a public issue for security disclosures.
