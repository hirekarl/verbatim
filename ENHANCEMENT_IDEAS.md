# Demo enhancement ideas (optional)

**Everything in this document is optional.** None of it is scheduled work, none of it is a commitment, and none of it blocks the Saturday Jul 18 demo — `TODO.md` and its Sprint 2 day-by-day plan remain the authoritative source of what's actually committed. Treat this as a menu, not a punch list.

## Why this doc exists

As of Jul 14, Sprint 2's architecture work (multi-agent split, native Anthropic SDK migration, Cloud Run deploy, Workspace Add-on) is done and verified end-to-end against a real Google Doc. That's real, substantial work — but it's all backend/infra. Functionally, Saturday's demo will look the same as Sprint 1's demo did: same sidebar, same category breakdown, same suggestion/comment flow. None of the architecture improvements are visible to someone watching the demo.

Karl wants to explore whether there's low-lift, genuinely demo-visible work we could pick up between now and Saturday — something the audience can actually *see* as new, beyond "we changed what's running underneath." A research pass across the evaluator/rule coverage, the open issue tracker, and the Add-on's sidebar UX turned up a few concrete candidates, cataloged below with rough owner splits and time estimates.

**This is a starting menu for Karl and Christina to talk through offline** — pick whatever's interesting, split however makes sense, or skip all of it if the remaining time is better spent on rehearsal and polish. Nothing here should come at the expense of Friday's dedicated demo-prep/rehearsal day.

One known gap is deliberately **excluded**: the #69 golden-edge category-tiebreaker bug. Two prior PRs (#70, #72) already tried and failed to fix it reliably; it's accepted at ~67% accuracy and deferred post-demo. Re-attempting it now is low-odds, not low-lift.
